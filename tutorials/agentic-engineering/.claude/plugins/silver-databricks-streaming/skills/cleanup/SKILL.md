### Step 2 — Stop and Delete All Databricks Pipelines (Both Targets)

The tutorial creates pipelines under **two** bundle targets (`dev` and `unified`). Both must be cleaned up — the `unified` target creates production-mode pipelines that auto-start and consume quota.

**Stop and delete pipelines from BOTH targets:**

```bash
source .env

# Stop + delete all dev-target pipelines
for PID in $(databricks bundle summary --target dev --output json 2>/dev/null | python3 -c "
import sys, json
s = json.load(sys.stdin)
for p in s.get('resources',{}).get('pipelines',{}).values():
    print(p['id'])
" 2>/dev/null); do
  echo "Stopping $PID..."
  databricks pipelines stop $PID 2>/dev/null
  echo "Deleting $PID..."
  databricks pipelines delete $PID 2>/dev/null
done

# Stop + delete all unified-target pipelines
for PID in $(databricks bundle summary --target unified --output json 2>/dev/null | python3 -c "
import sys, json
s = json.load(sys.stdin)
for p in s.get('resources',{}).get('pipelines',{}).values():
    print(p['id'])
" 2>/dev/null); do
  echo "Stopping $PID..."
  databricks pipelines stop $PID 2>/dev/null
  echo "Deleting $PID..."
  databricks pipelines delete $PID 2>/dev/null
done
```

Verify no tutorial pipelines remain:

```bash
databricks pipelines list-pipelines --output json | python3 -c "
import sys, json
pipelines = json.load(sys.stdin).get('statuses', [])
streaming = [p for p in pipelines if 'financial' in p.get('name', '').lower() or 'unified' in p.get('name', '').lower() or 'analytics' in p.get('name', '').lower()]
print(f'Related pipelines remaining: {len(streaming)}')
for p in streaming:
    print(f'  {p[\"name\"]}: {p[\"pipeline_id\"]}')
"
```

Verify:
```bash
databricks pipelines list-pipelines --output json | python3 -c "
import sys, json
pipelines = json.load(sys.stdin).get('statuses', [])

"
```

### Step 4 — Delete Databricks Jobs

Delete all 3 jobs:

```bash
# Get job IDs from bundle summary
databricks bundle summary --target dev --output json | python3 -c "
import sys, json
summary = json.load(sys.stdin)
for name, job in summary.get('resources', {}).get('jobs', {}).items():
    print(f'{name}: {job[\"job_id\"]}')
"

# Delete each job
for JOB_ID in $COMPLIANCE_JOB_ID $MARKET_REF_JOB_ID $REGULATORY_JOB_ID; do
  echo "Deleting job $JOB_ID..."
  databricks jobs delete $JOB_ID
done
```

### Step 5 — Drop Schemas and Tables (Both Dev + Unified)

Drop **both** schemas — the dev target and unified target use separate schemas:

```bash
# Drop dev schema (e.g., financial_risk_streaming_samlexrod)
databricks api post /api/2.0/sql/statements \
  --json '{"warehouse_id": "<WH>", "statement": "DROP SCHEMA IF EXISTS main.financial_risk_streaming_<username> CASCADE", "wait_timeout": "50s"}'

# Drop unified schema (e.g., financial_risk_unified_samlexrod)

  --json '{"warehouse_id": "<WH>", "statement": "DROP SCHEMA IF EXISTS main.financial_risk_unified_<username> CASCADE", "wait_timeout": "50s"}'
```

Verify both are gone:
```bash
databricks api post /api/2.0/sql/statements \
```

### Step 6 — Destroy AWS Infrastructure

Use Terraform to tear down the EC2 and associated resources:

```bash
cd infra
mise run destroy
```

This runs `terraform destroy` which will:
1. Terminate the EC2 instance (stops billing immediately)
2. Delete the OpenSearch domain (stops billing — takes ~10 minutes to fully delete)
3. Delete the security groups
4. Remove the IAM instance profile (if created)

Use `AskUserQuestion` to confirm before running `terraform destroy`.

Verify:
```bash
# Should show no resources
terraform -chdir=terraform show
```
